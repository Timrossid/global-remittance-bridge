# Security Policy

## 🛡️ Security Commitment
The Global Micro-Remittance Bridge project is committed to the security of its users and the integrity of the Stellar network. We take all reports of security vulnerabilities seriously.

## 🚩 Reporting a Vulnerability
If you discover a security vulnerability, please do **not** open a public issue. Instead, report it privately via:
- **Email:** security@global-remittance-bridge.example.com
- **Private Disclosure:** Use GitHub's private vulnerability reporting feature if available.

Please include:
- A detailed description of the vulnerability.
- Steps to reproduce (PoC).
- Potential impact.

## 🕒 Response Timeline
- **Acknowledgment:** We will acknowledge receipt of your report within 48 hours.
- **Triage:** We will provide an initial assessment of the severity within 5 business days.
- **Resolution:** We will work to patch the issue as quickly as possible before public disclosure.

## 📜 Disclosure Policy
We follow a coordinated disclosure process. We ask that you do not publicize the vulnerability until a patch has been released.
